Data Objects

Structured, Semi-structured, Unstructured

A data object is a collection of one or more data points that create meaning as a whole. The most common example of a data object is a data table, but others include arrays, pointers, records, files, sets, and scalar types (true/false).

Why do frameworks exist?

(hint: to simplify development)

User Interface (UI) Frameworks

Used to automate grid layout as well as design of common interface elements like buttons, forms, and menus.

Frontend Frameworks

Provide a skeleton to build your website around while still allowing flexibility with the final design. Important components to a front end framework include a gid layout system, conditional styling of elements, typography, and often include general layout elements seen in user interface (UI) frameworks.

Backend Frameworks

The role of a back-end framework is to allow your website to complete server-driven actions that can include connecting to remote databases, completing financial transactions, and other forms of complex, transactional business logic.
